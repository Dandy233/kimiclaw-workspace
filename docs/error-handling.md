# 错误处理和通知机制

## 概述

为解决"429或其他错误导致静默失败"的问题，已实施以下改进：

## 已实施的方案

### 1. 定时任务错误处理（✅ 已完成）

所有定时任务已更新，包含显式错误处理：

| 任务 | 重试策略 | 错误通知 |
|------|----------|----------|
| daily-it-news-search | 3次重试（30s→60s→90s间隔） | ✅ 失败时通知 |
| daily-it-news | 3次重试（30s→60s间隔） | ✅ 失败时通知 |
| daily-feishu-summary | 1次重试（30s间隔） | ✅ 无论成败都通知 |
| nightly-openclaw-update | 超时检测（4分钟） | ✅ 结果通知 |

### 2. 错误通知脚本（✅ 已完成）

位置：`scripts/error-notify.sh`
- 自动检测错误类型（429/超时/权限等）
- 发送飞书通知
- 可被其他脚本调用

### 3. 超时时间优化（✅ 已完成）

所有任务超时时间已调整：
- 搜索任务：400秒（足够完成深度搜索+重试）
- 发送任务：180秒
- 更新任务：360秒（6分钟）

## 如果仍遇到静默失败

如果问题仍然存在，请检查：

1. **查看日志**
   ```bash
   tail -f /tmp/openclaw/openclaw-$(date +%Y-%m-%d).log | grep -i error
   ```

2. **检查 cron 任务状态**
   ```bash
   openclaw cron list
   openclaw cron runs --jobId <jobId>
   ```

3. **手动测试消息发送**
   如果怀疑是飞书配置问题，可以测试发送
